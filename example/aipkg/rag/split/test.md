# 揭秘 Kubernetes（K8s）：容器编排的卓越之选

## 一、K8s 简介

Kubernetes，简称 K8s，是谷歌基于其内部多年容器编排经验开发并开源的容器编排系统。“K8s” 是 “Kubernetes” 首尾字母间有 8 个字母而得名。它能够自动化部署、扩展和管理容器化应用程序，极大地简化了大规模容器集群的管理工作。

## 二、核心特性

### 2.1 自动化部署与扩展

K8s 可依据预设规则自动部署应用程序的多个容器实例。当应用程序负载增加时，能自动增加容器数量以满足需求；负载降低时，减少容器数量，避免资源浪费。

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80
```



上述代码定义了一个 Nginx 应用的 Deployment，指定了要创建 3 个容器实例。

### 2.2 自我修复

K8s 会持续监控容器的运行状态。若某个容器崩溃，它会自动重启；若节点出现故障，会将容器调度到其他健康节点上，确保应用程序始终稳定运行。

### 2.3 服务发现与负载均衡

K8s 为一组容器提供统一的服务入口，通过内部的 DNS 系统实现服务发现。同时，能自动将请求均匀分配到各个容器实例上，实现负载均衡，提高应用程序的可用性和性能。

### 2.4 存储编排

支持挂载多种类型的存储卷，如本地存储、网络存储等。可以在容器重启时保留数据，方便数据的持久化存储和管理。

### 2.5 滚动更新与回滚

在进行应用程序更新时，K8s 支持滚动更新，即逐个替换容器实例，确保应用程序在更新过程中不间断运行。若更新出现问题，还能快速回滚到上一个稳定版本。

## 三、K8s 架构组件

### 3.1 控制平面（Control Plane）

- **API Server**：作为 K8s 的核心，提供 RESTful API 接口，是所有组件进行通信的入口。
- **Etcd**：分布式键值存储系统，用于存储 K8s 集群的所有配置信息和状态数据。
- **Controller Manager**：负责管理各种控制器，如节点控制器、副本控制器等，确保集群状态符合预期。
- **Scheduler**：根据节点资源使用情况和调度策略，将 Pod 调度到合适的节点上。

### 3.2 工作节点（Node）

- **Kubelet**：负责与控制平面通信，管理节点上的容器生命周期，确保容器按照 Pod 定义运行。
- **Kube - Proxy**：实现服务的网络代理和负载均衡，确保集群内的服务能够相互通信。
- **Container Runtime**：负责运行容器，常见的有 Docker、Containerd 等。

### 3.3 架构整体分层

K8s 采用**主从架构（Master/Node）**，也常被称为控制平面（Control Plane）和数据平面（Node），核心是 “控制平面管理全局，节点执行具体任务”。



### 3.4 kube-apiserver

**核心作用**：所有操作的统一入口，是 K8s 组件间通信的 “网关”。无论是 kubectl 命令、其他组件（如 scheduler/controller），还是外部系统，都必须通过 API Server 与集群交互。

**关键特性**：提供 RESTful API、身份认证 / 授权、数据校验、请求限流，是唯一能直接操作 etcd 的组件。




### 3.5 etcd

**核心作用**：K8s 的 “数据库”，存储集群所有的配置和状态数据（比如 Pod 定义、节点信息、服务规则）。

**关键特性**：分布式、强一致性、高可用的键值存储，集群的 “唯一真相源”，一旦 etcd 数据丢失 / 损坏，集群就会失效。

### 3.6 kube-scheduler

**核心作用**：“调度器”，负责为新创建的 Pod 选择最合适的 Node 节点运行。

**调度逻辑**：

1. 筛选：排除不满足条件的节点（比如资源不足、节点不可用）；
2. 打分：对符合条件的节点按优先级排序（比如资源利用率低、离存储近的节点得分高）；
3. 选择：选中得分最高的节点，通过 API Server 把 Pod 调度到该节点。

### 3.7 kube-controller-manager

**核心作用**：“控制器管理器”，运行一系列核心控制器，确保集群状态始终符合用户期望（比如 Pod 挂了自动重启、副本数不足自动补）。

**包含的核心控制器**：

- 节点控制器（Node Controller）：监控节点状态，节点宕机时标记并驱逐其上的 Pod；
- 副本控制器（Replication Controller）：维护 Deployment/StatefulSet 的副本数；
- 端点控制器（Endpoint Controller）：关联 Service 和 Pod，生成 Endpoints 数据；
- 命名空间控制器（Namespace Controller）：管理命名空间的生命周期。

### 3.8 cloud-controller-manager（可选）

**核心作用**：对接云厂商（如阿里云、AWS、腾讯云）的 API，实现 “云原生” 功能，比如自动创建负载均衡、动态分配云盘、同步云厂商的节点状态。

**特点**：把云厂商相关逻辑从 kube-controller-manager 中剥离，让 K8s 核心组件与云厂商解耦。



## 四、应用场景

### 4.1 微服务架构

K8s 非常适合微服务架构，能够轻松管理大量的微服务容器，实现服务之间的解耦和独立部署，提高开发和部署效率。

### 4.2 混合云与多云环境

支持在不同的云提供商和本地数据中心之间部署和管理容器化应用，实现混合云与多云环境下的资源统一管理。

### 4.3 持续集成与持续部署（CI/CD）

可与 CI/CD 工具集成，实现应用程序的自动化构建、测试和部署，加快软件交付速度。

## 五、总结

Kubernetes 凭借其强大的功能和卓越的特性，成为了容器编排领域的事实标准。它为开发者和运维人员提供了一个高效、可靠的平台，能够轻松应对大规模容器集群的管理挑战，推动了云计算和容器技术的广泛应用。
