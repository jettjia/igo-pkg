请参考LlamaIndex的文本分块设计，用golang实现有四个策略「固定字符大小+文档标题结构+字符递归切片+语义分块」的分块器，要求如下：

### 核心要求
1. 文档结构适配策略：
   - 识别中文标题（格式：H1："1. 标题" / "一、标题"；H2："1.1 标题" / "（一）标题"；H3："1.1.1 标题"）；
   - 标题必须与其下属的段落内容合并在同一个分块，禁止标题和内容跨块切割；
   - 不同层级标题的内容独立分块（如H1下的H2内容，优先归属于H1的块，超出长度再拆分）。

2. 固定字符大小限制：
   - 可配置chunk_size（单个块最大字符数）、chunk_overlap（相邻块重叠字符数）；
   - 块长度优先满足标题语义完整，仅当标题+内容超出chunk_size时，才按语义拆分。

3. 字符递归切片规则：
   - 分隔符优先级：空行(\n\n) > 换行(\n) > 句号(。) > 感叹号(!) > 问号(？) > 分号(；) > 逗号(，) > 空格；
   - 优先按分隔符切割，无合适分隔符时，才按字符数硬切；

4. 语义分块策略：
   - 无标题的文本，按句子拆分后计算余弦相似度（TF-IDF），相似度≥0.5的句子合并为一个块；
   - 单个句子超出chunk_size时，按语义短语拆分（避免切断核心概念）。

5. 代码要求：
   - 包含完整的注释、测试用例（含带层级标题的中文文本）
   下面每个策略都是公用的
   - 是否移除URL和邮箱，有开关配置
   - 替换掉连续的空格、换行符和制表符，有开关配置
   - 有重叠逻辑，可以配置重叠百分比10-20，默认是10%

### 输出格式
1. 先给出代码实现思路（分步骤说明）；
2. 完整可运行的golang代码；
3. 测试用例及运行结果说明
